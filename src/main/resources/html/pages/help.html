DASHBOARD

Security – Individual stocks, ETFs, or index funds, represented by a ticker symbol (e.g., VTI, VTSMX, AAPL, MSFT)

Asset – Group of Securities that represent the same underlying ETF/fund/etc. (e.g., Total Stock Mkt Asset = Security VTI + Security VTSAX)

Principal – Dollars invested, which is the amount you initially paid for the current shares that you own. This amount goes down as you sell shares (in proportion to the original investment, which is not affected by the current Value). Principal is always positive or zero.

Cash – The dollar amount of cash needed to be invested in a given portofio to sustain the portfolio. Cash gets reduced as proceeds from the portfolio flow back to the investor (via dividend and sales) reducing the original invested cash. Cash can have a negative value (indicating a surplus in cash reserves from profit, which is not used to finance the current portfolio).

Price – The latest price per share of the Security (may lag by 10-15 min during market hours)
Note: During off-market hours, Quantum will use day’s closing price.

Shares – Number of shares of each Security that you currently own
Note: If you sold 100% of a Security, it will no longer be displayed in the Securities table.

Value – The current dollar value of a Security or Asset. [Price x Shares]

Unrlzd Gain – Unrealized gain is the profit/loss on all of your unsold Shares. This represents the capital gains that would be achieved if the Asset were sold at the current Price. [Value – Principal]
Note: A positive value represents profit, while a negative value represents loss.

Rlzd Gain – Realized gain is the profit/loss on the sold shares. This includes profit from capital gains and dividend payouts. [Value – Principal, at the time of sale]
Note: A positive value represents profit, while a negative value represents loss.

Total Gain – Total gain, realized and unrealized. [Unrlzd Gain + Rlzd Gain]

Target Ratio – User-defined target proportion of the Asset Value in relation to the total portfolio Value, as a fraction of 1 (e.g., a 25% target proportion is represented as a Target Ratio of 0.25).

Curr Ratio – The current actual proportion of the Asset Value in relation to the total portfolio Value, as a fraction of 1.

Delta – The difference between Target Ratio and Curr Ratio. This measures how far off each Asset’s current Value is from the targeted Value (as derived from the user-defined Target Ratio) and can help guide decisions on which Asset(s) to invest into to align closer to Target Ratio. [Curr Value – Target Value]

In the following example, you are over-invested in MSFT and under-invested in AAPL. To drive Asset Value toward the Target Ratio, your next investment would be in AAPL.
ASSET TARGET $$ CURRENT $$ DELTA = (CURR - TARGET)
AAPL $7,000 @ 70% $6,000 @ 60% -$1,000
MSFT $2,000 @ 20% $3,000 @ 30% $1,000
FB $1,000 @ 10% $1,000 @ 10% $0
Total $10,000 $10,000


Contribution – Allocates user-defined Contribution dollar amount across the current Assets for the next investment to drive towards rebalancing the Asset mix to align with Target Ratios.

Rlzd Gain YTD – The Rlzd Gain for the current calendar year, stating on Jan 1.

Tax YTD – using a pre-set tax % assigned by user, this is a calculation of the potential tax on the gains made year-to-date.

CHARTS

“Portfolio vs Benchmark” Chart

Comparison over time of (starting with the date of your first Transaction):

  • Your portfolio Value (labeled with the username entered to log into Quantum)
  • An equivalent benchmark portfolio, using a benchmark security, such as the VTI to represent the U.S. stock market.
    Note: The benchmark Security can be configured in the Manage tab.
  • An equivalent cash portfolio (representing the cash needed to finance your portfolio or the benchmark portfolio).
    Note: Cash values can be negative (see Cash definition); however, for simplicity the graph will display zero values for any negative Cash values.

You can click and drag across a region of a graph to zoom in on a specific timeframe. Double-click anywhere in the graph to revert the view to the full timeframe.

“Simulated Target Portfolio vs Benchmark” Chart

Simulated comparison over the last 5 years of:

  • A cash portfolio starting with an initial amount and adding an incremental amount daily.
    Note: The default initial amount is $10,000 and an incremental daily amount of $20.
  • An equivalent benchmark portfolio of investing the dollars in the cash portfolio into a benchmark security.
  • An equivalent target portfolio of investing a mix of Assets that aims to closely track the user-defined Target Ratios. The portfolio only performs the initial purchase and the daily incremental purchases, and does not perform any selling or rebalancing. The portfolio attempts to show what you can achieve with an initial investment followed by regular incremental investments into a mix of assets targeting the Target Ratios, instead of investing into a benchmark or keeping the money as cash.

TRANSACTIONS

Quantum supports 5 types of Transactions. Each Transaction entry requires you to enter the Date, Transaction Type, Shares, and Price.

TRANSACTION TYPE DESCRIPTION SHARES PRICE
BUY Purchase of a Security* Number of Shares purchased Price per Share at which the Security was purchased
SEL Sale of a Security* Number of Shares sold Price per Share at which the Security was sold
DIV Dividend paid to you for a Security you own Number of Shares owned at the time of the dividend payout Dollar per share that the dividend paid out
Note: You can use a finance website to get this information if needed (e.g., Yahoo Finance).
SPL Stock split of a Security you own Number of new Shares after the split representing 1 old Share before the split (e.g. if a stock splits 3 for 1, this value would be 3) Price per Share after the split
CNV Conversion of a fund from one level into another, resulting in a new number of owned Shares and Share Price, without changing the Value Number of Shares you own after the conversion Price per Share after the conversion

* Occasionally the broker will split a single Buy or Sell Transaction into multiple transactions. Each broker Transaction needs to be entered separately.

Add (+) to a add a new Transaction

  • You can enter transactions in any order, but Quantum will assign an ID to each transaction in the order that they were entered.
  • For BUY/SEL transactions, use the following dates in Transaction entry:
    • Robinhood: (Submitted vs Filled date) – use Filled date (this is the date displayed in the History log. The two dates will be identical unless you submit a transaction in off-hours, in which case the transaction will be queued until the market opens and the transaction can be executed.
    • If using the Trade Confirmation data: use Trade date (not Settlement date), which should match the Filled Date.

Remove (-) to permanently delete an existing Transaction

Modify (…) to modify the Price of an existing Transaction

  • The only field that can be modified in an existing Transaction is Price.
  • Timing of entering new Transactions:
    • If entering a Transaction into Quantum within the same day of the transaction, Robinhood will display the Price at 2 decimal places.
    • For better precision, you can revise the Price to 4 decimal places once available in Robinhood the following day. In Robinhood, go to “History”, select the Transaction. Within the expanded Transaction view, click “Download Trade Confirmation” to view the 4-decimal Price. Click on the (…) button of the Transaction in Quantum to revise the Price.
    • Also, note that the monthly/annual statements only display 2 decimals in the Price.

MANAGE

Securities

Add Assets and Securities here. First add the Assets, then add the Securities within an Asset. The Asset names can be anything you choose. The Security names must match the market ticker symbols.

Target Ratios

  • All entered Assets will appear in the Target Ratios panel.
  • Enter any numerical value in the Ratio Factor text boxes, such that the Assets are properly represented in the target proportion relative to one another. E.g., entering 1 for 2 Assets will give each Asset a 50% allocation; entering 200 for Asset A, and 100 for each of Assets B and C will allocate 50% to A and 25% to B and C each.
  • Hit Save

Settings

  • IEX Publishable Token: You need to set up an account with IEX Cloud, and enter the IEX publishable token in this field. Go to http://iexcloud.io to register.

  • Tax Rate: enter a decimal (<1.0) to represent the desired marginal tax rate at which the capital gains and dividends are expected to be taxed
    Note: You can use the tax rate based on your salary bracket, or a generic rate 25-30%.

  • Contribution Amount: The dollar amount of your next investment (either as one-time, or at a regular weekly/bi-weekly/monthly/etc. cadence)

  • Chart: Benchmark Symbol: A benchmark security against which you will compare your portfolio’s performance. E.g., if you want to benchmark against the U.S. stock market, you can use “VTI”. If not user-defined, Quantum will use “VTI”.

  • Chart: Simulated Target Initial Principal: The “Simulated Target Portfolio vs Benchmark” chart simulates a portfolio with an initial investment and regular daily increment investment. This parameter defines the initial dollar amount.

  • Chart: Simulated Target Incremental Principal: The daily incremental investment used in the “Simulated Target Portfolio vs Benchmark” chart.

  • Chart: Simulated Target Months: The time period of the Simulated Target chart, in terms of the most recent number of months. If 0, Quantum will use the max time period available.

  • Chart: Simulated Target Whole Shares: This option simulates in the “Simulated Target Portfolio vs Benchmark” chart the effect of buying partial vs whole shares. This option typically does not have a significant effect, and can be ignored for most purposes.

  • Hit Save to save any changes to “Settings”.

Backup

It is a best practice to back up your Quantum data regularly (e.g. once a quarter). Select the format (CSV / JSON) and export/save the file. If any technical issues are encountered with your laptop/application/lost data, etc., backup files can be useful in restoring the data.
Note: The CSV file can also be useful for data analysis in a spreadsheet application, such as Microsoft Excel.
